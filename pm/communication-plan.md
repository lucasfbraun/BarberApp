# Communication Plan

## Objetivo
Manter visibilidade do progresso do projeto de forma simples e executavel.

## Publico
- Usuario / dono do projeto
- Time de desenvolvimento futuro
- Stakeholders de negocio

## Cadencia
- Diario: status rapido de impedimentos e avancos
- Semanal: entrega realizada, riscos e proxima prioridade
- Por sprint: review com demonstracao da funcionalidade pronta

## Canal
- Workspace do VS Code
- Planilha de cronograma
- Arquivos Markdown do projeto

## Padrao de atualizacao
- O que foi concluido
- O que esta em andamento
- O que bloqueia
- O que sera feito em seguida

## Artefatos
- charter
- WBS
- cronograma
- risk register
- change log
- definicao de pronto
